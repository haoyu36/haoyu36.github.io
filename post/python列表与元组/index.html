列表与元祖

[TOC]

列表是 Python 内置的一种数据类型，是一种有序的集合，包含 0 个或多个对象引用的有序序列，没有长度限制，可自由增删元素，使用灵活

列表

概述

列表中不同的元素类型可以相同，也可以不同，还可以是其他数据结构。列表支持：索引，分片，加，乘，最大最小值，判定成员资格，计算长度，迭代等等

从操作方式上看，列表像是数组和链表的综合体，可以当作队列和栈使用

实现

Python中的列表是由对其他对象的引用组成的的连续数组。指向这个数组的指针及其长度被保存在一个列表头结构中

在 Python 的官方实现中，list 采用分离式技术实现的动态顺序表

  • 连续表(一块连续的存储区)保证在 O(1) 时间进行元素访问，并能维持元素的顺序
  • 分离式技术保证了能容纳任意多元素，并且在更换存储区时 list 对象的标识 (id) 不变
  • 创建空表时不分配元素存储区，需要加入元素时，系统第一次分配 4 个元素的存储区。扩容时每次增加的比率趋近 1.125 倍
  • 等比扩容保证尾端加入元素的 append 的平均时间复杂性为 O(1)

from timeit import timeit

count = 10**4

def test1():
    '''加号拼接列表'''
    lst = []
    for i in range(count):
        lst = lst + [i]

def test2():
    '''使用操作符+=替代+'''
    lst = []
    for i in range(count):
        lst += [i]

def test3():
    '''extend在列表末尾一次性追加另一个序列中的多个值'''
    lst = []
    for i in range(count):
        lst.extend([i])

def test4():
    '''append追加'''
    lst = []
    for i in range(count):
       lst.append(i)

def test5():
    '''列表生成式创建列表'''
    lst = [i for i in range(count)]

def test6():
    '''list函数可以将所有序列转换为列表'''
    lst = list(range(count))

In : timeit("test1()", "from __main__ import test1", number=100)
Out: 18.793210283998633

In : timeit("test2()", "from __main__ import test2", number=100)
Out: 0.1109811429923866

In : timeit("test3()", "from __main__ import test3", number=100)
Out: 0.15882780199171975

In : timeit("test4()", "from __main__ import test4", number=100)
Out: 0.10526732800644822

In : timeit("test5()", "from __main__ import test5", number=100)
Out: 0.04239512997446582

In : timeit("test6()","from __main__ import test6", number=100)
Out: 0.022145277995150536
  • 创建列表效率最高的是使用 list(range(count))，其次是使用列表推导式
  • Python 对操作符 += 进行了优化，直接修改原数据，而非新建对象。效率也高于列表方法 extend
  • append 方法用于在列表末尾追加新的对象。直接修改原来的列表
  • 直接使用操作符 + 对列表进行相加的效率是很低的，因为列表相加会返回新列表，会浪费大量内存和花费很多时间

pop 方法移除列表中的一个元素（默认是最后一个），也可以进行索引删除，并返回该元素的值。pop 方法是唯一一个既修改列表又返回值的列表方法

删除末尾的元素的效率是高于头部的。pop 的时间复杂度是O(1)，pop(n) 的时间复杂度为 O(n)

推导式用于快速构建列表，字典和集合。列表推导和生成器表达式，字典推导式都有自己的局部作用域

元组

列表与元组的区别

  • 列表是可变对象，元组是不可变对象(引用不可变)
    • 因此元组可以作为字典的键 (可 hash)
    • 当经常修改对象时，首选列表
  • 元组有着更小的字节数，性能略有提升
  • 元组是异构的，列表是同构的
    • 元组是不同种类的东西的序列，可以将元组作为一个连贯的单元来处理。元组的每个位置都具有其特定的意义，可作为记录或结构。比如用 (x, y) 记录坐标
    • 列表是相同类型的东西的序列，列表的顺序是很重要的，每个元素在功能上是等价的

元组的可变性

元组中保存的是对象的引用，如果引用的元素是可变的，即使元组本身不可变，元素依然可变。元组的不可变性指保持的引用不可变，与引用对象无关
