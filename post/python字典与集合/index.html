字典

概述

字典是内置类型中唯一的映射 (mapping) 结构，基于哈希表存储键值对数据。模块的命名空间，实例的属性和函数的关键字参数都使用到了字典，标准库中所有映射类型都是利用 dict 实现的

Out: {'a': 1, 'b': 2, 'c': 3}

字典的实现

概述

数据的存储与访问是计算机最基本的功能。数据访问的基本方式是基于存储位置，如果知道所需数据存储在哪，常量时间内就可以访问

一般来说，如果数据项连续存储，而关键码就是存储数据的地址(下标)，那么这时候基于关键码检索就能在 O(1) 时间内找到数据

如果关键码不能或不适合作为数据存储的下标时，可以通过一个变换把它们映射到一种下标，这样就把基于关键码的检索转变为基于整数下标的直接访问元素。这种将关键码映射到下标称为散列函数

内置类型字典就实现了基于关键字的存储与检索，字典是 Python 中唯一**映射（mapping）**结构

在 Python 官方实现中，字典基于散列表(hash table)技术实现，采用开放地址法解决冲突

在散列函数值分布均匀和装载因子不高的情况下（0.7），散列表字典检索、插入、删除操作的时间开销可以看作常量

散列函数

散列函数的选择会影响冲突的概率。常用的散列函数有除余法（整数关键码）和基数转换法（整数或字符串关键码）

字典可以存储任何类型的值（对象），但只有不可变对象（可散列的数据类型）才能作为键。如果字典的关键字可以变化，就可能造成虽然存在某些数据，但却找不到它们的情况，破坏了数据存储的规则和完整性

冲突消解

散列函数是从大集合到小集合的映射，必然会出现多个不同的关键码对应到同一下标位置的情况。表中的元素越多，出现冲突的可能性就越大

负载因子 = (散列表中当时实际数据项/散列表能容纳的元素个数)

不管采用那种消解技术，随着元素的增加，出现冲突的可能性也会增大。可以考虑扩大散列表的存储区，但扩大存储要付出重新分配存储区和再散列装入数据项的代价。即以空间换时间

python 字典实现

创建空字典或很小的字典时，初始分配的存储区可容纳 8 个元素，当 dict 对象在负载因子超过三分之二时自动更换更大的存储区，并把已经保存好的内容重新散列到新存储区里

如果当前字典对象不太大，就按当时字典中实际元素的 4 倍分配新存储区。当字典里的元素超过 50000 时，改为按实际元素的 2 倍分配新存储区

Python 会保证散列表中的表元有三分之一是空的，在快要达到这个阈值时，会将原有的散列表复制到一个更大的空间里

字典的特点

  • 字典在内存上开销巨大
  • 字典使用了散列表，而散列表必须的稀疏的，导致空间效率的低下
  • 键查询很快，字典的实现是典型的空间换时间，虽然有着巨大的开销，但提供了无视数据量大小的快速访问

str1 = "k:1|k1:2|k2:3|k3:4"
def str2dict(str1):
    dic = {}
    for iterm in str1.split('|'):
        key, value = iterm.split(':')
        dic[key] = value
    return dic

集合

集合存储非重复对象，非重复指除不是同一对象外，值也不能相等。集合常用于判定素是否包含在集合中，集合的元素必须是可哈希的对象

Python 内置集合类型有两种：

  • set()：一种可变的、无序的、有限的集合，其元素是唯一的、不可变的（可哈希的）对象
  • frozenset()：一种不可变的、可哈希的、无序的集合，其元素是唯一的、不可变的（可哈希的）对象
