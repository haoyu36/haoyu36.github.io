分层是计算机技术最核心的一个思想。计算机科学中的任何问题，都可以通过加上一层逻辑层来解决

网络层次#

[TOC]

为了解决 (兼容)不同设备之间的通信问题，需要一套通用的网络协议

一：分层#

1.1 网络为什么分层#

分层是为了解决复杂性，如果把所有的步骤都放到一个层次，那么这一层就会变得非常复杂，耦合度非常高，不容易理解

分层有很多好处：

  • 独立：各层之间相互独立，高层是不需知道底层是如何实现的，只需要通过接口调用即可
  • 实现：各层可以采用最适当的技术来实现，改变实现方式只需保持接口不变，方便协议维护与方便升级

在网络协议的分层中，不仅仅会根据功能划分层次，还会考虑层与层之间的封装与传递。在向下的过程中需要添加下层协议所需要的首部或者尾部，而在向上的过程中不断拆开首部和尾部

1.2 其他分层结构#

软件架构的分层：

  • 用户界面层
  • 业务逻辑层
  • 数据访问层

mysql的分层：

  • server 层
  • 引擎层

硬件的分层：

  • 半导体物理学
  • 晶体管开关，是建立在半导体物理学上的模型
  • 逻辑门，是建立在开关上的模型
  • CPU，是建立在逻辑门上的模型

语言的分层：

  • 机器语言
  • 汇编语言
  • 编程语言

二：分层模型#

哪几层#

模型异同比较#

计算机网络的分层模型有 OSI 的七层模型和 TCP/IP 的四层模型，现在实际使用的是综合的五层模型。他们的异同

相同点：

  • 都采用了层次结构
  • 都能够提供面向连接和无连接两种通信服务机制

不同点：

  • OSI 七层模型； TCP/IP 是四层结构
  • TCP/IP 将数据链路层和物理层合并为网络接口层
  • OSI 七层模型是理论模型，一般用于理论研究，其分层有些冗余； TCP/IP更实用一些

为什么 TCP/IP 去除了表示层和会话层#

表示层 (抽象传递信息的语法与语义)，会话层( 建立及管理会话,如验证用户登录)

由于会话层、表示层、应用层都是在应用程序内部实现的，最终产出的是一个应用数据包，而应用程序之间是几乎无法实现代码的抽象共享的，这也就造成 OSI 设想中的应用程序维度的分层是无法实现的，例如，我们几乎不会认为数据的压缩、加密算法算是一种协议，而会话的概念则更为抽象，难以用协议来进行描述，所以在后来的 TCP/IP 协议框架的设计中，便将表示层和会话层与应用层整合在一起

每层任务及数据单位、服务对象#

层次任务数据单位服务对象
应用层为用户提供数据传输服务，各种应用程序协议，如 HTTP、DNS报文应用程序
传输层接收来自应用层的数据，必要时切割成较小的数据单元，然后把数据单元传递给网络层报文段(TCP)或用户数据报(UDP)进程
网络层通过路由选择算法，为分组选择最适合的传输路径分组主机
数据链路层通过各种控制协议，将有差错的物理信道变为无差错的、能可靠传输数据帧的数据链路帧同一链路上的主机
物理层在传输媒介(电话线或光纤电缆)上传输数据比特流比特流
  • 应用层： 应用软件都是在应用层实现的，专注于为用户提供应用功能而不用关心数据是如何传输的。应用层是工作在操作系统中的用户态，传输层及以下则工作在内核态
  • 传输层：为应用层提供网络支持

数据如何在各层之间传输#

三：常见协议及端口#

  • 协议：控制两个对等实体进行通信的规则的集合。TCP/IP 是一个实现 Internet 体系结构的协议族
  • 端口：端口号是 16 位非负整数 (0～65535)

举例：

  • 应用层
    • HTTP：超文本传输协议，port：80
    • HTTPS：超文本传输安全协议，port：443
    • DNS：域名系统，port：53
    • RPC：远程调用协议
    • SMTP：简单邮件传输协议，port：25
    • FTP： 文件传输协议，port：20/21
    • SSH：安全外壳协议，port：22
    • DHCP：动态主机配置协议
    • LDAP：轻量级目录访问协议，port：389
  • 传输层
    • UDP：用户数据报协议
    • TCP：传输控制协议
  • 网络层
    • ICMP：Internet 控制报文协议
    • OSPF：开放式最短路径优先
  • 链路层
    • ARP：地址解析协议
    • STP：生成树协议

四：网络设备#

网络包从客户端计算机发出之后，一般要经过集线器、交换机和路由器最终进入互联网。家用路由器已经集成了集线器和交换机的功能

设备功能缺点层次
集线器连接计算机等网络终端，将数据包传输到下一个转发设备会将数据广播到所有电脑，当连接的电脑较多时会导致严重的安全性和可靠性问题物理层
交换机连接同一个网络下不同的主机设备，通过交换学习记录每个电脑的 mac 地址生成转发表(有过期时间)，所以不会发生碰撞多台交换机连接形成的拓扑结构容易出现环路问题，这时数据包会不停的转发，占满带宽。使用 STP 协议可以将有环路的图变成没有环路的树链路层
路由器连接两个或多个局域网，当一个网络包送到路由器时，会根据路由表来决定如何转发网络层

路由器的分组转发流程:

  1. 从 IP 数据包中提取出目的主机的 IP 地址，找到其所在的网络
  2. 判断目的 IP 地址所在的网络是否与本路由器直接相连，如果是，则不需要经过其它路由器直接交付，否则执行 ③
  3. 检查路由表中是否有目的 IP 地址的特定主机路由。如果有，则按照路由表传送到下一跳路由器中，否则执行 ④
  4. 逐条检查路由表，使用每一行的子网掩码与目的IP匹配。若找到匹配路由，则按照路由表转发到下一跳路由器中，否则执行步骤 ⑤
  5. 若路由表中设置有默认路由，则按照默认路由转发到默认路由器中，否则执行步骤 ⑥
  6. 无法找到合适路由，向源主机报错。
